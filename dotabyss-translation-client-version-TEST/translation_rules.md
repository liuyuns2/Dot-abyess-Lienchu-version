# 遊戲文本在地化翻譯與校正指南

本文件是後續翻譯與校正遊戲劇情文本時必須遵守的規則。目標是將輸入文本（日文、簡中、簡轉繁或混合文本）轉化為高品質、自然流暢的台灣繁體中文。

## 最高優先級：系統與格式保留

- Key 值絕對不修改。JSON 的日文 Key、代碼、看似錯誤或奇怪的 Key 都必須保持原樣，除非使用者明確要求修 Key。
- 只修正 value。所有翻譯、潤飾、校正都只能動中文譯文 value。
- 控制碼與標記必須完美保留，包括但不限於 `<br>`、`<user>`、`<size=48>`、變數、顏色標籤、符號、數字、換行標記等。
- 控制碼不得漏刪、錯置、改寫或移位到不合語意的位置。
- 成人文本只修正翻譯品質，使其自然流暢、術語前後一致。嚴禁自行擴寫、加料或刪減情節。

## 正式譯名表

- 專有名詞、角色名、稱號、地名與特定術語一律以 names 正式譯名表為準。
- 最高優先級字典（High-Priority Dictionary）路徑：
  `D:\dotabyss-translation\GITHUB-dotabyss-translation\-X-\name.json`
- 若 `name.json` 與本文件中的角色範例、過往口頭規則或腳本建議不一致，一律以 `name.json` 為準。
- 若文本中出現表內 Key 對應內容，必須嚴格使用該表 value 作為譯名。
- 腳本產生任何 dry-run 或 write 建議前，必須先套用 `name.json` 的正式譯名。
- `<user>` 必須保留為 `<user>`，不得翻譯。

## 台灣在地化語感與用詞

常見替換方向：

| 咱們 | 我們 |
| 這兒 | 這裡 |
| 哪兒 | 哪裡 |
| 啥 | 什麼 |
| 沒法 | 沒辦法 |
| 挺 | 很、蠻 |
| 幹活 | 工作 |
| 蹲點 | 埋伏 |
| 內鬼 | 內賊、臥底 |
| 崗位 | 職位 |
| 手辦 | 模型 |
| 妓院 | 娼館 |

### UI 與系統詞彙

若文本包含 UI 或系統介面，依台灣用語修正：

| 避免用語 | 台灣繁中建議 |
| --- | --- |
| 視頻 | 影片 |
| 軟件 | 軟體 |
| 屏幕 | 螢幕 |
| 設置 | 設定 |
| 默認 | 預設 |
| 充值 | 儲值 |

### 繁簡與錯字修正

- `复製` 改為 `複製`。
- `工資` 依語境改為 `薪水`。
- `這裏`、`那裏` 統一改為 `這裡`、`那裡`。
- 統一使用 `裡`，不使用 `裏`。
- 依語境區分 `聯繫` 與 `聯絡`；人與人之間互動一律用 `聯絡`。

## 打破翻譯腔

- 嚴禁硬照日文句序排列。
- 避免生硬直譯，例如「在某某中出征」、「露出來」、「一股腦」等不自然組合。
- 若原始簡中、日文譯文語意邏輯混亂，或二轉手造成嚴重翻譯腔，必須在不改變核心劇情的前提下大膽重組句子。
- 第一優先是台灣繁中自然語感，其次才是逐字對應。

## 標點符號與排版

- 中國式雙引號 `“”` 必須改為台灣標準引號 `「」`。
- 中國式單引號 `‘’` 必須改為 `『』`。
- 英文句點連打 `......` 統一改為標準全形刪節號 `……`。
- 清理簡中轉繁時產生的不必要半形空格。
- 保持文本緊湊、自然，不加入無意義空白。

## 角色語氣與人設

所有對話必須符合人物設定，避免每個角色都像同一個人說話。

### 羅莎

- 語氣正式、嚴謹。
- 帶有騎士精神與榮譽感。
- 避免俚俗或過度口語。

### 貝麗莎

- 語氣撒嬌、可愛、黏人。
- 可帶活潑語尾，但不可過度加料。
- 需要保留角色原本的親暱感。

### 瑪麗娜

- 語氣精明、市儈、圓滑。
- 具有商人的算計與社交手腕。
- 可保留圓滑拖腔，但不可翻成平淡直白句。

### 主角內心吐槽

- 必須自然、幽默。
- 像一般年輕人的內心 OS。
- 不使用過度書面或生硬文字。

## 執行提醒

- 翻譯與校正時，必須同時檢查語意準確度、繁簡轉換、台灣在地化、角色語氣、控制碼保留。
- 若遇到原文或既有譯文混亂，必須在不改變核心劇情的原則下重新潤飾，使其符合台灣繁中自然語感。
- 若是 JSON，輸出與修改時必須保持合法 JSON 格式。
- Key 不動，value 才是工作範圍。
